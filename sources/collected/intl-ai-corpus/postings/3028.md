About the Role

This role focuses on building and evolving a modern CI/CD
ecosystem that enables fast, reliable, and secure delivery
of AI agents and cloud‑native applications. You’ll work at
the intersection of DevOps, AI delivery, and platform
engineering, partnering closely with developers and cloud
teams across regions. Your work will reduce friction in the
development lifecycle while raising the bar on automation,
security, and reliability.

What You’ll Do

- Design build and deployment pipelines for AI agents and
  Azure‑based containerized applications.
- Automate CI/CD workflows to reduce manual effort across
  build, test, and release processes.
- Implement GitHub Actions workflows, including secure
  authentication, secrets management, and self‑hosted
  runners.
- Enable reliable deployments to Kubernetes environments using
  Helm and AKS.
- Integrate CI/CD pipelines with artifact repositories,
  security tools, and third‑party systems.
- Collaborate with AI developers, infrastructure teams, and QA
  to continuously improve delivery practices.

Technologies You’ll Work With

- GitHub Actions, Job DSL, GitHub & GitLab APIs
- Docker, Kubernetes, Helm, Azure Kubernetes Service (AKS)
- Azure (ACR, Entra ID, Key Vault), GitHub Secrets
- Python, Bash, Groovy
- JFrog Artifactory, HashiCorp Vault, SonarQube
- Ansible, Terraform

What You’ll Bring

- 5+ years of experience in CI/CD, DevOps, or platform
  engineering roles.
- Strong expertise in Github Actions pipelines.
- Hands-on experience building CI/CD workflows for
  containerized and Kubernetes‑deployed applications.
- Proficiency in Bash scripting and automation, with working
  knowledge of Python.
- Operational experience with Azure services, including AKS,
  ACR, Entra ID, and secrets management.
- Working knowledge of Helm and Kubernetes deployment patterns
  across multiple environments.
- Ability to collaborate effectively with distributed teams
  across time zones.

Bonus Skills

- Experience with additional CI/CD platforms such as Azure
  Pipelines, GitLab CI/CD, or AWS CodePipeline.
- Familiarity with LLMOps practices, AI agent architectures,
  and multi‑environment promotion models.
- Understanding of GitOps workflows using tools like ArgoCD or
  Flux.
- Experience with Go for CI/CD tooling or automation.
- Relevant certifications such as CKA, AZ‑400, or AWS DevOps
  Engineer Professional.
