### Raisonnement classification (LLM upstream)

This role focuses on designing and implementing user
interfaces FOR AI-powered applications, not building or
deploying AI models. The work involves creating UX patterns
for AI features, building front-end components that consume
AI APIs, and designing interaction patterns. While it
requires understanding AI capabilities, the core work is
front-end development and design, not hands-on model
development or deployment.

Job Description

As a UX/UI Engineer, you will bridge design and engineering
to create AI-enhanced user experiences that are both
powerful and approachable.

Design & User Experience:

- Design and prototype user interfaces that make complex AI
  capabilities accessible and intuitive for users
- Create design systems and component libraries that
  standardize AI interaction patterns across products
- Conduct user research to understand how people interact with
  AI features and iterate based on behavioral insights
- Design conversational interfaces, prompt builders, and
  feedback mechanisms for AI-powered features
- Develop information architecture for applications
  incorporating semantic search, recommendations, and AI-
  generated content
- Create responsive, accessible designs that meet WCAG 2.1 AA
  standards and Section 508 compliance

Implementation & Development:

- Transform designs into production-ready code using modern
  front-end frameworks (React, Vue, Angular)
- Build reusable UI components that elegantly handle AI states
  (loading, streaming responses, error handling)
- Implement real-time interfaces for AI features including
  chat interfaces, live previews, and progressive disclosure
- Develop interactive visualizations for AI model outputs,
  confidence scores, and decision explanations
- Create smooth animations and micro-interactions that provide
  feedback during AI processing

AI Integration & Innovation:

- Leverage AI design tools (Figma AI, Framer AI, Galileo AI)
  to accelerate design iteration and prototyping
- Use AI coding assistants to rapidly implement and test
  design variations
- Design and implement A/B testing frameworks for AI feature
  adoption and usability
- Create intuitive controls for AI parameters (temperature,
  context, model selection) accessible to non-technical
  users
- Develop patterns for graceful degradation when AI services
  are unavailable
- Design transparency features that help users understand AI
  decision-making

Collaboration & Process:

- Partner with AI/ML engineers to understand technical
  constraints and possibilities
- Work with product managers to define AI feature requirements
  and success metrics
- Collaborate with content strategists to develop AI
  interaction copy and guidelines
- Conduct design reviews focusing on AI-human interaction
  patterns
- Document design decisions, AI interaction patterns, and
  component usage guidelines
- Mentor team members on designing for AI-augmented
  experiences
