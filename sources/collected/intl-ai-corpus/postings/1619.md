### Raisonnement classification (LLM upstream)

This is a QA/test automation role where AI tools are used to
assist with testing tasks (test generation, failure pattern
analysis, code review, RAG for documentation lookup), but
the role does not involve building, training, fine-tuning,
or deploying AI/ML models. The primary work is test
automation and quality engineering, with AI being an
assistive tool rather than the core focus.

## Responsabilités

- Analyze requirements, user stories, and API contracts to create comprehensive test scenarios and automation strategies
- Build and maintain automated tests using Selenium, Playwright, and Cypress with clean, reusable frameworks (POM, BDD)
- Test APIs thoroughly including authentication, payloads, error handling, and contract validation using Postman and REST automation
- Write SQL queries to validate data integrity, transformations, and business logic in databases
- Integrate test suites into CI/CD pipelines and monitor execution quality across browsers and platforms
- Use AI-powered tools and RAG approaches to extract knowledge from technical docs, speed up code reviews, and recognize defect patterns
- Track defects in JIRA with clear reproduction steps and impact analysis while collaborating with developers on code reviews
- Maintain test documentation, dashboards, and quality metrics while participating in Agile ceremonies
